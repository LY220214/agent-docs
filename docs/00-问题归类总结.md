### 难度星级是什么意思？

文档里每个分类都标了星级（⭐到⭐⭐⭐⭐⭐），这不是随便标的，而是根据"一个刚入行的开发者能不能理解"来判断的：

| 星级 | 含义 | 举个例子 |
|------|------|----------|
| ⭐⭐ | 基础概念，梳理过就能理解 | "项目背景介绍"——只要梳理过项目，就能说清楚 |
| ⭐⭐⭐ | 进阶概念，需要深入理解 | "架构思维怎么体现？"——你得真理解什么是架构，不能只背定义 |
| ⭐⭐⭐⭐ | 高难度，需要实战经验 | "上下文压缩策略"——光看过文章不够，得自己做过或者深入分析过 |
| ⭐⭐⭐⭐⭐ | 顶级难度，需要深度理解+设计能力 | "Agent Loop怎么实现？"——需要能画出架构、写出伪代码、说清权衡 |

### 怎么把这份文档当学习清单用？

每个知识点前面都有个复选框 `- [ ]`，你可以这样用：

1. **第一遍扫读**：快速过一遍所有知识点，把"完全没概念"的标出来
2. **第二遍精读**：对照文档映射表，找到对应文档，逐个攻克
3. **第三遍自测**：遮住解释，看自己能不能流畅讲清每个知识点

建议用支持Markdown的编辑器（比如VS Code + Markdown Preview Enhanced插件），这样勾选复选框的状态会自动保存。

### 遇到看不懂的术语怎么办？

这份文档里会出现"Agent Loop""MCP""Subagent"之类的术语。如果你看到某个词觉得陌生，去 [术语速查表](./README.md#术语速查表) 查一下，每个术语都有通俗的类比解释。比如"Agent Loop"在那里被解释成"做菜的流程：尝一口→觉得淡→加盐→再尝一口"，比干巴巴的定义好懂多了。

## 🏗️ 第一大类：项目架构与技术实现

### 💡 初学者理解要点

这一类知识点讲的都是"一个 AI Agent 项目到底是怎么回事"。核心是搞清楚你有没有真正动手做过东西，以及做的时候有没有想清楚为什么这么做。

**核心概念通俗解释**：

- **Agent Loop**：就像一个在不断解决问题的机器人。它的工作方式是：思考当前情况→决定下一步做什么→执行→检查结果→如果没完成就继续思考。跟人修电脑一样：先看症状→猜测原因→换个零件试试→开机检查→没修好就换个思路再来。
- **Subagent协作**：想象一个公司里，老板（主Agent）把任务拆成几份，分给不同部门的员工（Subagent）。每个员工只管自己那块，做完把结果交回给老板。老板再综合所有人的结果，做出最终判断。
- **AI代码修复建议**：不是AI凭空猜答案，而是像医生看病一样：先看症状（日志）→查病历（知识库）→开处方（生成修复代码）。每一步都有依据，不是瞎蒙。

## 🤖 第二大类：Claude Code技术深度

### 💡 初学者理解要点

Claude Code是Anthropic公司出的一个AI编程助手，你可以把它理解成一个"住在终端里的AI程序员"。这一类知识点考的是你对这个工具的理解有多深：是只会用它写代码，还是能看懂它的内部设计？

**核心概念通俗解释**：

- **MCP协议**：全称Model Context Protocol，翻译过来就是"模型上下文协议"。你可以把它想象成USB接口：不管什么设备，只要符合USB标准就能插上电脑用。MCP也一样，不管什么工具（数据库、搜索引擎、文件系统），只要符合MCP标准，AI就能调用它。没有MCP之前，每接一个新工具都得写一套专门的对接代码；有了MCP，就像有了统一插座，插上就能用。
- **SDK底层原理**：SDK就是"软件开发工具包"，别人把复杂的功能封装好了，你直接调用。Claude Code SDK的底层，本质上是一个"调度中心"：它管理着AI什么时候思考、什么时候调用工具、什么时候把结果返回给你。理解底层原理，就是理解这个调度中心是怎么运转的。
- **思维链修正**：AI有时候会"想歪了"，思维链修正就是让AI把推理过程一步步写出来，如果中间某步走偏了，可以及时发现并纠正。就像数学考试写解题步骤，老师能看出你哪一步算错了，而不是只看最终答案对不对。
- **Skills系统**：Skills就是AI的"技能包"。每个Skill解决一类特定问题，比如"代码评审""写测试""查文档"。AI面对一个任务时，会从技能库里挑出合适的Skill来用。问题在于：技能太多，AI会犯选择困难症；技能太少，又不够用。怎么平衡，就是这一类知识点的核心。

## ⚙️ 第三大类：系统设计与工程实践

### 💡 初学者理解要点

前两类讲的是"做了什么"和"用的工具是什么"，这一类讲的是"怎么把东西做出来、做得靠谱"。它关注的是工程能力：不是能不能写代码，而是能不能设计一个真正能跑在生产环境里的系统。

**核心概念通俗解释**：

- **SDK重构**：官方的Claude Code SDK是个"全家桶"，什么功能都塞进去了，导致启动慢、体积大、不好嵌入别的系统。重构就是把不需要的部分砍掉，只保留核心能力，就像把一辆房车改装成跑车：去掉厨房和床，只留发动机和方向盘，跑得更快。
- **上下文压缩**：AI能"看到"的信息是有限的（就像人的短期记忆容量有限）。对话越长，历史信息越多，AI就"看不过来"了。上下文压缩就是用更少的文字表达同样的意思，就像把一部两小时的电影浓缩成三句话的剧情简介，核心信息不丢，但篇幅大幅缩短。
- **高可用设计**：系统不可能永远不出故障。高可用设计就是让系统在出问题时还能基本运转，就像手机信号不好时自动从5G切到4G，网速慢了但至少还能用。

## 💻 第四大类：AI Coding与Skills系统

### 💡 初学者理解要点

AI Coding就是"用AI来写代码"。这一类知识点关注的不是你会不会用AI，而是你对"人+AI协作写代码"这件事有没有深入思考过：AI擅长什么、不擅长什么、人和AI怎么分工最合理。

**核心概念通俗解释**：

- **人写代码 vs AI写代码**：AI写代码就像用导航开车，大部分路它能帮你规划好，但遇到复杂路况（模糊的需求、特殊的业务逻辑）还是得人来判断。最好的模式不是"全交给AI"或"全自己写"，而是人负责想清楚要什么，AI负责快速实现，人再负责检查质量。
- **Skills管理**：AI的技能库就像一个工具箱。工具太少，遇到问题没合适的工具用；工具太多，找工具的时间比干活的时间还长。怎么让AI快速找到对的工具，又不被太多选项搞晕，是Skills系统设计的核心难题。

## 🎯 第五大类：个人经历与软技能

### 💡 初学者理解要点

技术学习不只学技术，还要学会把做过的事情讲清楚、有自己的思考、对职业有规划。这类知识点看似简单，其实很多人做不好，因为平时没想过怎么梳理自己的经历。

**学习建议**：

- **项目经历梳理**：别列流水账，用2分钟讲清楚"做了什么、怎么做的、结果如何"。想听的是故事，不是清单。
- **架构思维**：不是背设计模式的名词，而是从全局角度看问题：系统由哪些部分组成、各部分怎么配合、为什么这么设计而不是那么设计。
- **职业规划**：不需要说"我想当CTO"这种大话，但要有方向感：你对AI的哪个方向感兴趣、在朝那个方向做什么努力。
